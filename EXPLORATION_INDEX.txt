💡 KEY INSIGHTS
═════════════════════════════════════════════════════════════════════════════════

DESIGN PATTERNS:
├─ State: useState for local, Zustand for global, Context for app
├─ Forms: Blank object template, set() helper function
├─ Dialogs: Portal wrapper, controlled visibility
├─ Database: Transaction pattern for batch operations
├─ Currency: Paise storage (×100), formatted display (÷100)
├─ Animations: Animated API with stagger, timing, spring
└─ Positioning: Safe area insets for notch-safe placement

STYLING:
├─ Theme-aware: All colors from theme.colors
├─ Material Design 3: Follow MD3 principles
├─ Consistent: Primary for main, tonal for secondary, text for utility
└─ Responsive: Works on all screen sizes, light/dark mode

DATABASE:
├─ SQLite: Synchronous queries in transactions
├─ Naming: user_id links all records to user
├─ Timestamps: ISO format with nowISO() function
├─ IDs: Generated with newId() function
└─ Relations: Foreign keys link related tables
